# Custom Fields

The **Custom Fields** section allows you to define additional data fields that are collected during member registration. These fields appear on the member signup form and can be used to gather information specific to your business needs.

---

## Overview

Custom fields let you collect extra information beyond the standard member profile. Common examples include emergency contact details, allergies, t-shirt sizes, or any other data relevant to your organization.

---

## Creating a Custom Field

Click **Add New Custom Field** to create a new field. The following options are available:

### Field Name
A descriptive label shown to members on the registration form (e.g., "Emergency Contact Name").

### Field Key
A unique identifier used internally (e.g., `emergency_contact`). This key is used in exports and internal references.

### Field Type
The type of input displayed to the member:
- **Text** - A standard text input
- **Checkbox** - A true/false toggle
- **Date** - A date picker
- **Select** - A dropdown with predefined options

### Options
Only shown when **Field Type** is set to **Select**. Define the dropdown options available to the member.

### Display Order
Controls the order in which custom fields appear on the registration form. Lower numbers appear first.

### Show on Class Roster
When enabled, this field's value will be displayed on class rosters, making it visible to trainers and staff during classes.

### Required Field
When enabled, members must fill out this field to complete their registration.

---

## Managing Custom Fields

Custom fields appear in a table showing their name, key, type, order, and settings. You can edit or delete fields as needed.

> **Note:** Changing or removing a custom field does not affect data already collected from existing members.
